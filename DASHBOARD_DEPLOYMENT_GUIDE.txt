METHOD 1: VERCEL DEPLOYMENT (RECOMMENDED - FREE & EASY)
========================================================

STEP 1: Setup GitHub Repository
------
1. Create GitHub account (free): https://github.com/signup
2. Create new repository: Click "New Repository"
   - Name: "thermal-power-plant-dashboard"
   - Make it "Private" (for security)
   - Click "Create repository"

STEP 2: Upload Dashboard Code
------
1. Clone repository to your computer:
   $ git clone https://github.com/[YOUR-USERNAME]/thermal-power-plant-dashboard.git
   $ cd thermal-power-plant-dashboard

2. Create project structure:
   $ npm create vite@latest . -- --template react
   $ npm install recharts lucide-react

3. Replace the default App.jsx with the progress_dashboard.jsx code provided

4. Create .env.local file (for security):
   VITE_APP_TITLE="Progress Dashboard"
   VITE_API_URL="https://your-api.com" (optional, for future API integration)

5. Update package.json with:
   ```json
   {
     "name": "thermal-power-plant-dashboard",
     "private": true,
     "version": "1.0.0",
     "type": "module",
     "scripts": {
       "dev": "vite",
       "build": "vite build",
       "preview": "vite preview"
     },
     "dependencies": {
       "react": "^18.0.0",
       "react-dom": "^18.0.0",
       "recharts": "^2.10.0",
       "lucide-react": "^0.263.1"
     },
     "devDependencies": {
       "@vitejs/plugin-react": "^4.0.0",
       "vite": "^4.3.9"
     }
   }
   ```

STEP 3: Push to GitHub
------
1. Add files:
   $ git add .

2. Commit:
   $ git commit -m "Initial dashboard setup"

3. Push:
   $ git branch -M main
   $ git push -u origin main

STEP 4: Deploy to Vercel
------
1. Go to https://vercel.com/signup (create free account)

2. Click "Import Project"

3. Select "Import Git Repository"
   - Paste your GitHub repository URL
   - Select "Import"

4. Configure project:
   - Framework: Vite
   - Root Directory: ./ (default)
   - Build Command: npm run build (default)
   - Output Directory: dist (default)

5. Add Environment Variables:
   - VITE_APP_TITLE = "Progress Dashboard"
   - Click "Deploy"

6. Wait for deployment (2-5 minutes)

7. GET YOUR PUBLIC URL:
   - Copy the URL shown after deployment completes
   - Example: https://thermal-power-plant-dashboard.vercel.app

✅ DASHBOARD NOW LIVE AT YOUR URL!

For Production, Implement One Of:

OPTION 1: Firebase Authentication (Recommended)
===============================================
1. Create Firebase project: https://firebase.google.com

2. Install Firebase:
   $ npm install firebase

3. Update App.jsx with Firebase config:
   ```javascript
   import { initializeApp } from "firebase/app";
   import { getAuth } from "firebase/auth";

   const firebaseConfig = {
     apiKey: process.env.VITE_FIREBASE_API_KEY,
     authDomain: process.env.VITE_FIREBASE_AUTH_DOMAIN,
     projectId: process.env.VITE_FIREBASE_PROJECT_ID,
     storageBucket: process.env.VITE_FIREBASE_STORAGE_BUCKET,
     messagingSenderId: process.env.VITE_FIREBASE_MESSAGING_SENDER_ID,
     appId: process.env.VITE_FIREBASE_APP_ID,
   };

   const app = initializeApp(firebaseConfig);
   const auth = getAuth(app);
   ```

4. Add environment variables in Vercel dashboard:
   - VITE_FIREBASE_API_KEY
   - VITE_FIREBASE_AUTH_DOMAIN
   - etc.

5. Replace LoginPage with Firebase auth:
   ```javascript
   const handleLogin = async (email, password) => {
     try {
       await signInWithEmailAndPassword(auth, email, password);
       // User logged in
     } catch (error) {
       console.error("Login failed:", error);
     }
   };
   ```

OPTION 2: Auth0 (Enterprise Solution)
=====================================
1. Create Auth0 account: https://auth0.com
2. Create application in Auth0 dashboard
3. Install Auth0 React SDK:
   $ npm install @auth0/auth0-react

4. Wrap App with Auth0Provider
5. Use useAuth0 hook for login/logout

OPTION 3: HTTP Basic Authentication (Simple)
=============================================
Vercel supports .htaccess style protection:
1. Use Vercel's built-in "Protected Routes" feature
2. Configure in vercel.json:
   ```json
   {
     "headers": [
       {
         "source": "/(.*)",
         "headers": [
           {
             "key": "x-robots-tag",
             "value": "noindex"
           }
         ]
       }
     ]
   }
   ```

3. Or use Vercel Password Protection:
   - Go to project settings
   - Enable "Preview Deployment Protection"
   - Set password

   - Column D: Progress %

   ```javascript
   useEffect(() => {

     const url = `https://docs.google.com/spreadsheets/d/${sheetId}/export?format=csv`;
     
     fetch(url)
       .then(res => res.text())
       .then(data => {
         // Parse CSV and update projectData
         setProjectData(prev => ({
           ...prev,
           dailyActivities: parseCSV(data)
         }));
       });
   }, []);
   ```

4. Dashboard auto-updates from Google Sheet!

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "Dashboard not loading"
SOLUTION:
1. Check internet connection
2. Clear browser cache (Ctrl+Shift+Del)
3. Try different browser
4. Check Vercel deployment status

ISSUE: "Login fails"
SOLUTION:
1. Verify email/password correct
2. Check caps lock
3. Try resetting credentials
4. Verify Vercel env variables set

ISSUE: "Data not saving"
SOLUTION:
1. Check browser localStorage enabled
2. Clear cache & cookies
3. Try private/incognito window
4. Check browser console for errors

ISSUE: "Mobile view broken"
SOLUTION:
1. Check Vercel build logs
2. Test on different phone
3. Clear phone cache
4. Try WiFi instead of mobile data

ISSUE: "Need to reset data"
SOLUTION:
1. Go to browser DevTools (F12)
2. Application → LocalStorage
3. Find dashboard URL entry
4. Delete the data
5. Refresh page

SECURITY BEST PRACTICES
================================================================================

✅ DO:
- Use HTTPS always
- Store passwords securely (never in code)
- Use environment variables for secrets
- Enable two-factor authentication (GitHub, Vercel)
- Regular security audits
- Restrict access to private repos
- Backup data regularly
- Update dependencies monthly

❌ DON'T:
- Share credentials via email/Slack
- Commit secrets to Git
- Use weak passwords
- Leave dashboard open on public computer
- Ignore security warnings
- Use default/demo credentials in production
